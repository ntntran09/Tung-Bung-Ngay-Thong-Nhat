# Repo Hygiene

## Purpose

This document explains which files are source of truth, which files are Godot-generated, and which repo quirks deserve care during cleanup or refactors.

## Canonical Source Files

Treat these as primary source artifacts:

- `*.gd`
- `*.tscn`
- `*.tres`
- `project.godot`
- `export_presets.cfg`
- `MAIN/dialogues/*.txt`
- art/audio assets under module `assets/`
- `README.md`
- `AGENTS.md`
- Markdown files under `docs/`

## Godot-Managed Files

### `.import`

- currently tracked in the repo
- regenerated by Godot when asset import metadata changes
- should usually stay in sync with imported source assets

Do not bulk-edit these by hand.

### `.uid`

- currently tracked in the repo
- used by Godot to maintain resource identity
- should travel with the corresponding script or resource

Do not delete these casually.

## Local Or Generated Files

### `.godot/`

- local editor/import cache
- ignored by `.gitignore`
- safe to delete when a full re-import is needed

### `EXPORT/`

- export output directory
- ignored by `.gitignore`
- should be treated as generated output, not source

If `EXPORT/` contents appear in a worktree diff, remember that they are build artifacts.

## Tracked Editor Backup Files

The repo currently contains tracked Godot temporary scene backups such as:

- `*.tmp`

Examples exist under:

- `MAIN/scenes/`
- `MAIN/scenes/Characters/`
- `CO_GANH/scenes/`
- `O_AN_QUAN/scenes/`
- `TRON_TIM/scenes/`
- `TRON_TIM/animations/`

Guidance:

- do not use `*.tmp` files as source of truth
- do not edit them as part of feature work
- remove them only in an intentional cleanup change after confirming nobody relies on them

## Current Repo-Specific Notes

### Shared pause scene path text is stale in some scene headers

The actual file on disk is:

- `MAIN/scenes/ui/game_control.tscn`

Some scene resources still record:

- `res://MAIN/scenes/game_control.tscn`

Godot can still resolve this via resource UID today, but the path text is stale and should not be copied into new work blindly.

### Dialogue files are part of export configuration

`export_presets.cfg` includes:

- `MAIN/dialogues/*.txt`

If dialogue files move or are renamed, update export presets as well as scene bindings.

### The backend URL is hard-coded

`MAIN/script/game_data.gd` contains the base URL for backend-dependent features.

That is easy to change locally and easy to forget in docs, so review related docs whenever it changes.

## Commit Guidance

### Documentation files

- `README.md`, `AGENTS.md`, and `docs/**/*.md` should be visible to `git status` when changed.
- If a documentation file does not appear in `git status`, check local ignore rules with `git check-ignore -v <path>`.

### Good small-scope changes

- scene/script changes plus the docs that explain them
- asset changes plus the matching `.import` updates
- one intentional cleanup category at a time

### Risky changes that deserve extra care

- deleting or regenerating many `.import` files
- deleting many `.uid` files
- bulk-removing tracked `*.tmp` files
- moving shared scenes used across modules
- renaming hub dialogue files

## Encoding Guidance

This repo contains Vietnamese text and some Windows-generated filenames. When editing dialogue or docs:

- prefer UTF-8
- confirm Godot still reads the dialogue files correctly
- avoid unnecessary renames of existing assets unless you are prepared to update all references

## Update Triggers

Update docs when any of these change:

- `project.godot`
- autoload list
- `MAIN/dialogues/`
- export filters
- shared pause UI location
- module entry scenes
